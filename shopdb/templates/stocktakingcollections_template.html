shop.db stocktaking

Please read the following information carefully

  • If a product does not exist in this list, but does exist, it has not yet been entered in the system. Make sure that the product is created before this stocktaking is entered and that the replenishment is also entered before it is entered.
  • Make sure all offline lists of purchases are entered in the system before you enter the stocktaking!
  • Make sure that all invoices are entered in the system!
  • Please make sure that no products are purchased during the stocktaking. If a product no longer exists, i.e. the count is 0, it will be marked as inactive automatically. To avoid this, you can check "Keep active?". When entering the stocktaking in shop-db2-admin you can then take this into account.
  • The field "Theoretical Stock" indicates how many products should theoretically be there. Please check carefully if there are any deviations in the count or if you have miscounted!
